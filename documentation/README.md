## Adding New Documentation

When creating new `.md` files:

1. **Identify the category** - Determine which subcategory folder is most appropriate
2. **Use existing folders** - If a logical category exists, use it
3. **Create new folders** - If no logical category exists, create a new subcategory folder
4. **Update this README** - Add the new category to this file if you create a new folder

## File Naming Convention

- Use descriptive, uppercase filenames with hyphens: `FEATURE-NAME.md`
- Be consistent with existing naming patterns
- Group related files in the same category folder
